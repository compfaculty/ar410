## Інструкція: встановлення та запуск activness-reporter

*English: [HOWTO.md](HOWTO.md)*

Цей документ для **кінцевих користувачів**, які хочуть завантажити готовий
виконуваний файл, налаштувати доступ і запустити програму. Встановлювати
інструментарій Rust не потрібно.

---

## 1. Завантаження програми

1. Відкрийте сторінку [Releases](https://github.com/compfaculty/ar410/releases).
2. Оберіть останню версію (наприклад, `v0.1.0`).
3. Завантажте архів для вашої операційної системи:
   - **Windows**: `activness-reporter-x86_64-pc-windows-msvc.zip`
   - **Linux**: `activness-reporter-x86_64-unknown-linux-gnu.tar.gz`
   - **macOS (Intel)**: `activness-reporter-x86_64-apple-darwin.tar.gz`
   - **macOS (Apple Silicon)**: `activness-reporter-aarch64-apple-darwin.tar.gz`

4. Збережіть файл у будь-яку зручну теку (наприклад, `Downloads/`).

---

## 2. Розпакування архіву

### Windows

1. Клацніть правою кнопкою по завантаженому `.zip` і оберіть **Extract All…**
   («Розпакувати все…»), або скористайтеся PowerShell:

   ```powershell
   Expand-Archive -Path .\activness-reporter-x86_64-pc-windows-msvc.zip -DestinationPath .\activness-reporter
   cd .\activness-reporter
   ```

2. У розпакованій теці має з’явитися `activness-reporter.exe`.

### Linux / macOS

У терміналі:

```bash
mkdir -p activness-reporter
tar -xzf activness-reporter-*.tar.gz -C activness-reporter
cd activness-reporter
```

Має з’явитися файл програми з ім’ям `activness-reporter`.

На деяких системах треба дозвіл на виконання:

```bash
chmod +x activness-reporter
```

---

## 3. Налаштування середовища (облікові дані)

`activness-reporter` очікує облікові дані Activeness у вигляді **змінних
середовища**. Почніть з **методу 1** (флаг `.env`); інші методи — за
потреби.

Приклад значень для файлу `.env` або ручного введення:

```env
ACTIVENESS_EMAIL=your@email
ACTIVENESS_PASSWORD=your_password
# Optional: override API URL (default: https://activeness.social/api)
# ACTIVENESS_API_URL=https://activeness.social/api
```

### Метод 1 — файл `.env` (рекомендовано)

Підходить, якщо ви на Windows і ще не звикли до термінала.

1. Відкрийте **Блокнот** (або інший **простий текстовий** редактор — не Word).
2. Вставте три рядки з прикладу вище. Замініть `your@email` на свій email
   Activeness і `your_password` на пароль. Рядки, що починаються з `#`,
   залиште як є, якщо вам не потрібен інший адрес API.
3. Оберіть **Файл → Зберегти як…**
4. У діалоговому вікні відкрийте **ту саму теку**, де лежить
   `activness-reporter.exe` (Windows) або `activness-reporter` (Linux/macOS).
5. У полі **Ім’я файлу** введіть точно: `.env` (крапка, потім літери env).
6. У полі **Тип файлу** (Windows) оберіть **Усі файли (`*.*`)**, щоб система
   не перейменувала файл на `.env.txt`.

**Порада (Windows):** у Провіднику файлів увімкніть **Вид → Показати → Розширення
імен файлів**, щоб бачити, чи файл випадково не зберегся як `env.txt` замість
`.env`.

### Метод 2 — лише для поточного вікна термінала

Якщо не вдається створити `.env`. **Після закриття термінала значення зникають.**
Для щоденної роботи зручніший метод 1.

1. Відкрийте термінал у теці програми (див. [розділ 4](#4-перший-запуск)).

**PowerShell** (часто на початку рядка є `PS`):

```powershell
$env:ACTIVENESS_EMAIL = "your@email"
$env:ACTIVENESS_PASSWORD = "your_password"
```

**Command Prompt (CMD)** (класичне чорне вікно; у рядку зазвичай немає `PS`):

```bat
set ACTIVENESS_EMAIL=your@email
set ACTIVENESS_PASSWORD=your_password
```

У CMD **не** ставте пробіли навколо знака `=`.

### Метод 3 — вікно «Змінні середовища» у Windows (розширено)

Значення зберігаються в параметрах **вашого облікового запису** (або для всього
комп’ютера, якщо додано *системні* змінні). Використовуйте лише якщо розумієте,
що дані залишатимуться в Windows, доки їх не видалити.

1. Відкрийте **Параметри → Система → Про програму → Додаткові параметри системи**
   (або введіть у пошуку Windows «змінні середовища»).
2. Натисніть **Змінні середовища…**
3. У блоці **Змінні середовища користувача** натисніть **Створити…** і додайте
   `ACTIVENESS_EMAIL`, потім повторіть для `ACTIVENESS_PASSWORD`. *(Системні
   змінні діють для всіх користувачів і зазвичай потребують прав адміністратора;
   для особистих логінів краще використовувати змінні користувача.)*
4. **Закрийте всі вікна терміналу**, потім відкрийте **нове**, щоб програма
   побачила змінні.

### Примітки

- `ACTIVENESS_EMAIL` і `ACTIVENESS_PASSWORD` мають відповідати вашому обліковому
  запису Activeness.
- Замість `ACTIVENESS_PASSWORD` можна використовувати `ACTIVENESS_PSW`, якщо
  так налаштовано у вашому середовищі.
- Рядок `ACTIVENESS_API_URL` не змінюйте, якщо вам не потрібна адреса API, відмінна
  від стандартної.

## 4. Перший запуск

### Windows

Дотримуйтесь порядку кроків при першому запуску.

#### Крок 1 — відкрити термінал у теці програми

Оберіть **один** спосіб.

**Найпростіше (Провідник):**

1. Відкрийте теку, де лежать **`activness-reporter.exe`** і файл **`.env`**.
2. Один раз клацніть по **рядку адреси** (де показано шлях).
3. Введіть `powershell` і натисніть **Enter**, щоб відкрити PowerShell у цій теці,
   **або** введіть `cmd` і натисніть **Enter** для командного рядка.

**Вручну:**

1. Відкрийте **Windows Terminal**, **Windows PowerShell**, **PowerShell 7** або
   **Командний рядок**.
2. Перейдіть до теки командою `cd`, наприклад:

   ```powershell
   cd D:\Downloads\activness-reporter
   ```

   Якщо ви в **CMD**, а програма на **іншому диску** (наприклад `D:`), виконайте:

   ```bat
   cd /d D:\Downloads\activness-reporter
   ```

#### Крок 2 — з’ясувати, який саме оболонкою ви користуєтесь

- **Windows Terminal** — лише вікно: у кожній **вкладці** все одно PowerShell або
  CMD — використовуйте відповідні команди нижче.
- **PowerShell:** на початку рядка зазвичай є **`PS`**. Перед ім’ям програми
  **обов’язково** введіть **`.\`** (така вимога безпеки).
- **CMD:** рядок виглядає як `C:\...>`. Можна ввести ім’я `.exe` **без** **`.\`**.

#### Швидка підказка

| Що відкрито | Як запускати програму |
|-------------|------------------------|
| PowerShell (у теці програми) | `.\activness-reporter.exe` |
| CMD (у теці програми) | `activness-reporter.exe` |
| Linux / macOS (у теці програми) | `./activness-reporter` |

#### Крок 3 — запуск інтерфейсу TUI (за замовчуванням)

**PowerShell:**

```powershell
.\activness-reporter.exe
```

**Командний рядок (CMD):**

```bat
activness-reporter.exe
```

Запускається TUI (текстовий інтерфейс) для перегляду та обробки цілей Activeness.

#### Інші типові команди

**Фоновий режим** (для планувальників / автоматизації):

```powershell
.\activness-reporter.exe --headless
```

```bat
activness-reporter.exe --headless
```

**Оновлення профілів браузера / входу на сайтах:**

```powershell
.\activness-reporter.exe --update-profile
```

```bat
activness-reporter.exe --update-profile
```

**Локальний JSON із цілями** (без Activeness API; для тестів):

```powershell
.\activness-reporter.exe --from-json .\targets.json
```

```bat
activness-reporter.exe --from-json targets.json
```

### Linux / macOS

З теки, де лежать `activness-reporter` і `.env`:

```bash
./activness-reporter
```

Фоновий режим:

```bash
./activness-reporter --headless
```

Оновлення профілів:

```bash
./activness-reporter --update-profile
```

Запуск з локальним JSON:

```bash
./activness-reporter --from-json targets.json
```

## 5. Перший запуск і Playwright

Програма використовує автоматизацію браузера (через Playwright). Під час
**першого** запуску може статися таке:

- Завантаження бінарних файлів браузера
- Підготовка профілів браузера на диску

Це може зайняти більше часу й місця на диску лише **перший** раз; наступні
запуски зазвичай швидші.

Якщо перший запуск триває довго — це нормально.

---

## 6. Усунення несправностей

- **Нічого не відбувається / програма одразу завершується**
  - Переконайтеся, що `.env` лежить у **тій самій** теці, що й виконуваний файл.
  - На Windows перевірте ім’я файлу: має бути `.env`, а не `.env.txt` чи
    `env.txt` (увімкніть показ розширень у Провіднику).
  - Перевірте правильність `ACTIVENESS_EMAIL` і `ACTIVENESS_PASSWORD`.

- **`activness-reporter` / команда не знайдена (PowerShell у Windows)**
  - У **PowerShell** запускайте `.\activness-reporter.exe` **з теки**, де лежить
    `.exe`, **з** префіксом `.\` на початку.

- **Помилки автентифікації або API**
  - Переконайтеся, що обліковий запис Activeness активний і дані введені вірно.
  - Для власного бекенду задайте `ACTIVENESS_API_URL`.

- **Проблеми з відображенням у терміналі (TUI)**
  - Скористайтеся сучасним терміналом.
  - На Windows краще підходять Windows Terminal або PowerShell, ніж застарілий
    `cmd.exe` з точки зору сумісності.

- **Автоматизація браузера не працює**
  - Переконайтеся, що дозволені вихідні HTTPS-з’єднання.
  - Корпоративне ПЗ безпеки іноді блокує автоматизовані браузери.

Якщо проблема лишається, збережіть повний текст з терміналу і надішліть його
людині, яка обслуговує ваше розгортання Activeness.
